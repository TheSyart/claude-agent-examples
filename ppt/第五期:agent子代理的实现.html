DECISION 10 · GUARDRAILS
这版实现的重点
是可控，而不是并发
教程里的子代理是串行同步执行：主循环等待它完成。这更适合解释上下文隔离和工具白名单。
Plan State
全量覆盖
每次传完整 todos，避免增量补丁合并歧义。
Sub Tools
工具白名单
子代理不能再派子代理，也不改主 todo。
Loop Limit
max_turns
防止工具试错无限循环。
Main Context
摘要回传
压缩执行轨迹，只留结论。
组件s07s08
工具数45
任务状态显式 todo沿用主状态

CHOICE 12 · ENGINEERING CALLS
把对的取舍
提前钉死
教学版能跑就行；工程版要可控、可观测、防越权。每条决策都对应一处运行时兜底。
CORE
先想清楚再动手 —— 后期重构成本，远高于一次想清楚的成本。
PLAN
工具而非子代理
计划要可动态调整，子代理一返回就冻结。
SUBAGENT
预设身份
researcher 只读 / general 全工具，可测可审。
WHITELIST
写在代码里
tool_names 在 registry.py；模板只放身份文案。
TODO STATE
完成即清, 未完则推
stop_reason 时校验残单：未办妥回推 history 续跑, 办妥则重置 TODOS。
