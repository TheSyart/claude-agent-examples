// 问题  ·  一问一答太慢了
问题定义 · 被动响应的局限
复杂任务
需要用户一直催
s01-s11 的 Agent 都是被动问答：用户说一句，Agent 回一句。多步骤任务需要用户不断跟进，容易丢上下文。
1
步骤多
列出文件 → 读取 → 分析 → 写报告
2
上下文长
每轮都要重复目标和已完成的步骤
3
进度难跟踪
用户无法直观看到整体进度
传统对话
用户：帮我整理 import
Agent：先列出文件？
用户：好
Agent：文件有这些...
用户：提取每个文件的 import
...

理想：用户只说一次目标，Agent 自己跑完
// 方案  ·  目标树
Goal Tree
把大目标
拆成一棵树
每个目标有 id、content、status、parent_id、depends_on、success_criteria。Agent 用 `update_goals` 维护这棵树，system prompt 会实时注入当前目标状态。
根
root
高层目标描述
子
children
拆解后的子目标
赖
depends_on
前置依赖全部完成才能开始
update_goals 示例
{
  "id": "count_lines",
  "content": "统计每个 .py 文件行数",
  "status": "pending",
  "parent_id": "root",
  "depends_on": ["list_files"],
  "success_criteria": "得到每个文件的行数"
}
状态流转：pending → in_progress → completed / failed。
// 流程  ·  自主模式
/goal 命令
用户说一次
Agent 跑到底
用户输入 `/goal 统计 src 下所有 .py 文件行数`，AgentLoop 进入 `_run_goal_mode()`，循环推进直到完成或达到上限。
1
创建 root goal
把用户目标作为根节点
2
重建 system prompt
注入当前目标树
3
循环执行
每轮调用 runner.step()，检查目标状态
AgentLoop._run_goal_mode
for turn in range(max_autonomous_turns):
  reply = runner.step(history)
  if "DONE:" in reply:
    goal_store.complete(root_id)
  if not goal_store.has_pending_or_in_progress():
    break
  history.append({
    "role": "user",
    "content": "继续推进目标..."
  })
