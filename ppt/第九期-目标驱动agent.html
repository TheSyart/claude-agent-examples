// 第 2 页  —  为什么需要 Goal
先解决一个老问题
说“做完了”
不等于真的做完
就像装修队说“已经装好了”，你还是会看墙面、开关和水管。Agent 也一样，要用检查结果说话。
没有标准做到哪算哪，容易提前结束。
写清标准检查通过以后，才算完成。
记
不听口头保证，只看能不能检查通过。
// 第 3 页  —  怎么把目标说清楚
不用写一大篇
一个目标
说清 3 件事
说结果，讲边界，再给一个能看到结果的检查办法。
1
最后要看到什么说结果，不用替 Agent 写每一步。
2
哪些能做，哪些不能动把范围和不能碰的地方讲清楚。
3
怎么确认已经完成给出能运行、能看到结果的检查办法。
例
/goal 只读统计当前目录里的所有 Python 文件，列出行数，并用可复核命令证明哪个最长

// 第 5 页  —  什么时候停下来
停下来也要有理由
自动结束
只有 2 种结果
完成检查通过，或者碰到必须由你决定的事情。
01
做完了
计划中的事情做完，所有检查都通过。
02
需要你决定
缺权限、缺信息，或者下一步风险太高。
没有预设轮数上限；用户可随时按 Ctrl+C 退出。
