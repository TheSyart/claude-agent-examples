CLOSE 05 · COMPLETION GUARD
模型说"办完了"
不算真办完
仅靠模型自报完成靠不住 —— 它常在还有 pending 任务时就给最终回答。需要在 stop_reason != tool_use 这一刻，由代码再校一遍 todolist，再决定结束还是继续。
①
校验未完成项
filter status != completed
②
未办妥就回推
把残单当作 user 消息塞回 history，continue 内层循环
③
办妥则重置
TODOS = []，下一轮会话从空白开始，不污染新差事
